# 日常冒險系統規格

## 定位

日常冒險不是第四種路徑玩法，而是承載不同路徑玩法的長期收集循環。

玩家選擇同行的小貝狗或已收集的小日獸與冒險地點。關卡托盤只會帶入玩家已收集的該地點路線拼圖；完成路徑關卡後，再從經過地點的內容池取得文字事件、日常漫畫格或照片。

## 第一版多頁流程

1. `/game/daily`：首次進入時先顯示三步 onboarding；完成後回到日常冒險首頁，顯示目前進度、同行夥伴、地點拼圖與收藏摘要。
2. `/game/daily/prepare`：選擇同行夥伴與最多兩個冒險地點，並顯示各地點已收集的拼圖張數。
3. `/game/daily/stages`：選擇已解鎖關卡，查看關卡使用的路徑模式與首通獎勵。
4. `/game/daily/play`：實際路徑玩法。
5. `/game/daily/result`：先以主線共用的事件對話 UI 演出遭遇，再顯示事件／漫畫／照片與首通地點拼圖。第一關若同行者是小貝狗，事件對話結束後會先進入該地點場景的小貝狗拍照環節，拍完才顯示結果；收下照片後，冒險收藏會新增一張獨立的小貝狗實拍照片。
6. `/game/daily/collection`：查看已收集的事件、漫畫與照片。

## 首次導引串接

第一章完成後的導引順序固定為：

1. 小貝狗答應後，進到小麥房間，完成首次可遊玩的回家 Hub 三功能教學（日記 / 小日獸 / 睡覺）。
2. 播放大廳開放 modal。
3. 進到大廳後，以 highlight tooltip 鎖定「日常冒險」卡，引導玩家先點日常冒險。
4. 日常冒險首頁在這個導引狀態下會直接帶玩家進入 Level 1；完成 Level 1 後，結果頁主按鈕改為「回到大廳」，並寫入 `hasCompletedDailyAdventureLobbyGuideLevelOne`。這個旗標獨立於一般 `completedStageIds`，避免測試或重玩紀錄讓導引短路。
5. 回到大廳後，以 highlight tooltip 鎖定「繼續旅程」主線卡；玩家點擊後接到 `scene-daily-adventure-return-room`，回小麥房間播放「上次在便利商店遇到的小日獸，被他跑了，下次遇到我要好好拍下來」→ 小貝狗「嗷」→ 睡覺隔天 → `scene-morning-hub` → 安排路線。點擊繼續旅程時才標記 `hasSeenDailyAdventureMainStoryReturnGuide`，整段導引才算完成。
6. 這段回主線會補齊既有青蛙線索流程的進度：`hasTriggeredWorkLunchForgotBentoEvent = true`、`streetForgotLunchFrogPhotoAttemptCount >= 1`、`hasSeenFirstFrogReturnHomeScene = true`。因此隔天從 `scene-morning-hub` 出門時會走既有 `/game/arrange-route?storyRoute=frog-clue`，目標是「街道：傳單吹走」，不是一般 `day=next` 行程。

## 進度模型

- 編號關卡控制教學節奏、盤面複雜度與路徑模式解鎖。
- 收藏單位是「單張路線拼圖」，不是整個地點。`捷運・直路`、`捷運・寬轉窄`、`捷運・窄轉寬`、`捷運・寬路`是四個不同收藏項目。
- onboarding 說明每日遊玩會遇到文字事件、日常漫畫與拍照機會，並在最後展示玩家取得的三張初始地點拼圖。
- 新玩家的三張初始拼圖是 `捷運・直路`、`便利商店・寬轉窄`、`街道・窄轉寬`。每張仍是獨立收藏；取得一張街道拼圖不會自動開放其他三種街道接頭。
- 小貝狗永久可選；其他小日獸依主線收集狀態開放。
- 首次通關提供一張固定路線拼圖；重複遊玩時會優先掉落已選地點尚未收集的其他接頭拼圖。
- 抽取內容時優先補未收集項目，避免連續冒險沒有收藏進度。

## 路徑模式

### 寬窄接路

地點拼圖與無地點道路都可能出現。玩家需要配合起點、終點及相鄰拼圖的寬窄接頭完成路線。

- 地點拼圖只使用 `public/images/route` 內已準備好的完整圖，不以場景圖、圖示與文字自行合成。
- 街道、捷運、便利商店與早餐店目前各有四種寬窄圖；公車站只有直路圖。
- 一般地點與道路拼圖在同一盤只能各使用一次；只有標示 ∞ 的轉彎拼圖可以重複取用。
- 公園目前只有捷運出口模式圖，因此不會被偽裝成寬窄接路拼圖；若所選地點沒有該關可用的既有圖，出發時會要求更換地點。

### 無限轉彎

直接沿用主線既有的無限轉彎關卡元件與規則：轉彎拼圖可以重複拖放；點擊已放置的拼圖時，相鄰轉彎拼圖會一起轉向；每次關卡有四次轉向限制，並保留原本的接點判定、教學、接合動畫與出發轉場。

### 捷運出口

多出口、機關與路網探索模式。目前仍在開發設計中，在關卡選擇頁只顯示預告，不列為可完成關卡。

## 狀態邊界

日常冒險狀態使用獨立的 `moment:daily-adventure:v3` localStorage。v3 將收藏層級從地點改為單張路線拼圖：

- 已選夥伴與地點
- 已完成關卡
- 已收集路線拼圖 ID
- 已收集事件／漫畫／照片
- 進行中的冒險與最後一次結果

玩家金幣、體力與主線解鎖狀態仍由既有 `PlayerProgress` 管理。這樣可以避免把可重複冒險的內容狀態繼續堆進主線進度模型。

測試期間，每次進入日常冒險首頁會把體力補到至少 100，方便連續驗證關卡。

## 第一關設計基準

第一關是直向一格的寬窄拼圖：

- 上方是寬路終點、下方是窄路起點，中間只有一個可放置格。
- 托盤只提供三張有地點的拼圖，不混入一般道路：`捷運・直路`、`便利商店・寬轉窄`、`街道・窄轉寬`。
- 玩家需辨認上下接頭，選出能接通路線的 `便利商店・寬轉窄`。

## 第二關設計基準

第二關沿用原第一關的 Figma `走走小日 2026 / daily_1`（node `11364:15045`）盤面規格，不沿用舊版 Daily 的直向簡化棋盤：

- 上方是固定的 `wide_to_wide` 終點與旗幟，下方是固定的 `start_home_narrow` 起點。
- 起終點之間是 2×3、共六個可放置格；不能縮成主線無限轉彎關卡原本的兩個空格。
- 托盤顯示一張可無限重複使用的轉彎拼圖，以及第一關提供的三張單次使用地點拼圖；各地點拼圖依自己的上下寬窄接頭參與連通判定。
- 點擊已放置的轉彎拼圖時，自己與正交相鄰的轉彎拼圖一起旋轉；拖曳、移動、移除與接合後鎖定沿用主線互動。
- 連通判定不使用 Daily 目前的直向 `topEdge / bottomEdge` 比對。要抽用主線 `ArrangeRouteView` 的四向 connector、`NEIGHBOR_MAP`、精確接頭比對與 BFS 起終點連通判定。
- 轉彎圖、拖曳預覽、連動旋轉、接合動畫與出發轉場抽用主線現有實作；Daily 只提供盤面尺寸、起終點、可用地點拼圖與完成回呼。
- 畫面保留 Figma 的盤面、盤面下方規則提示、托盤區、左側提示按鈕與右側出發按鈕，header 改為 `level 2`，不自行替換成另一套 Daily 關卡 chrome。
