# Figma UI 落地規則

這份文件整理 `moment_prototype` 在使用 Figma 設計稿時的落地原則。目的不是讓實作「照稿切圖」，而是先判斷設計稿在遊戲流程中的正確位置，再決定要如何做進現有 UI。

若設計稿與遊戲流程理解衝突，**以遊戲內承載層級與互動脈絡為優先**。

---

## 核心原則

### 1. 先判斷這張稿是什麼層級

開始實作前，先回答這張稿屬於哪一類：

- 獨立 page
- `GameFrame` 中間手機舞台內容
- overlay / modal
- 某個區塊元件
- 單一卡片 / 按鈕 / 小組件

若這一步沒有先判斷，最容易把「設計稿上的容器」誤做成新頁面。

### 2. 先找它屬於哪個既有流程

在本專案裡，新的 UI 通常不是獨立存在，而是掛在既有流程上。實作前要先回答它屬於：

- `ArrangeRouteView`
- `GameSceneView`
- `GameFrame`
- 某個 event modal

例：

- 安排行程中的地圖，屬於 `ArrangeRouteView`
- 通勤事件中的一段說明，通常屬於 event modal
- 劇情中間的角色介紹，通常屬於 `GameSceneView`

### 3. Figma 先拿來看「位置關係」，不是直接整張照搬

設計稿通常可直接參考的是：

- 主要資訊層級
- 元件之間的位置關係
- 比例
- 留白
- 視覺分組

不應直接照搬的常見部分：

- 為了展示而重畫的一整套 header / footer
- 已由現有遊戲容器提供的外框
- 不符合當前互動狀態的靜態裝飾

例：

- 若設計稿中的地圖被畫在手機畫面裡，代表它通常是「手機舞台中的內容」或「覆蓋在手機舞台中的 overlay」
- 這不代表要另外再做一份頁面級的 header / 出發按鈕

### 4. 先沿用現有容器，再放入新內容

優先順序：

1. 先找現有承載容器
2. 再決定新內容怎麼放進去
3. 最後才調樣式細節

避免做法：

- 先把 Figma 畫面整頁切出來
- 實作完成後才回頭思考它到底應該掛在哪個流程

### 5. 先做「最小正確承載」，再做視覺細修

正確順序應該是：

1. 放對地方
2. 尺寸對齊
3. 互動掛對流程
4. 再補資訊與 polish

不要先追求畫面完整度，卻把承載層級做錯。

## 實作前檢查清單

收到 Figma 稿後，先問自己：

1. 這是 page、舞台內容、overlay，還是元件？
2. 它屬於 `ArrangeRouteView`、`GameSceneView`、`GameFrame`，還是 event modal？
3. 設計稿裡哪些是內容本身，哪些只是展示容器？
4. 專案裡有沒有相似的現成承載方式可以沿用？
5. 我是不是正要把設計稿的外層容器也一起做進去？

若第 5 題答案是「是」，應先停下來重判層級。

---

## 專案中的具體判斷規則

### A. 手機尺寸畫面不等於新頁面

如果 Figma 看起來像一支手機中的畫面，先假設它是：

- `GameFrame` 的中間內容，或
- 蓋在中間內容上的 overlay

而不是新 route。

### B. 有既有頁面 header / footer 時，不要複製第二份

如果新 UI 是在既有流程中打開：

- 不要重做同一套頁面標題
- 不要重做原本已存在的底部主按鈕
- 除非那份 header / footer 是新容器內部獨有的內容

### C. 安排行程相關新 UI，優先考慮 overlay

對 `ArrangeRouteView` 來說，若新內容是：

- 臨時查看資訊
- 補充理解地點 / 任務 / 規則
- 不離開當前排路線脈絡

優先考慮做成 overlay / modal，而不是 route 跳轉。

### D. 書頁 / 日記背景先當成裝飾層，不要把它誤當內容容器

如果設計稿裡有像日記、書頁、紙張、裝訂線這類背景：

- 先把它視為背景裝飾層
- 不要讓背景圖本身決定內容的置中方式
- 不要另外再墊一層白底或邊框去「模仿」背景

正確做法通常是：

1. 書頁 section 先對齊外層真正應該貼齊的邊
2. 背景圖貼在該 section 裡，保留需要露出的上邊 / 左邊 / 裝訂區
3. 內容另外包一層 inner wrapper
4. 用 inner wrapper 的 padding 或 offset 避開背景裝飾區

避免做法：

- 因為背景圖看起來像一本書，就把內容整塊做成視覺置中
- 用外層 padding 把整個 section 卡住，導致明明應該貼右卻貼不過去
- 書頁背景已經有紙張邊緣了，還再加額外白底、邊框、假書頁

實務上，若左邊是裝訂線區，內容應該：

- 用單獨內層包住
- 整體往右排
- 左側保留固定留白給背景

而不是把內容直接在整張書頁上做幾何置中。

## 本次案例：地圖

### 錯誤方向

- 把 Figma 中的地圖做成獨立 page
- 把設計稿裡示意性的 header / 出發按鈕也一起做進地圖

### 正確方向

- 地圖屬於 `ArrangeRouteView`
- 技術上應做成蓋在當前頁面上的手機尺寸 overlay
- 設計稿中的 header / 出發按鈕只作為位置參考，不應在 overlay 中重做

## 本次案例：小日獸圖鑑書頁

### 錯誤方向

- 把書頁背景當成內容卡片，再疊白底或 border
- 讓外層容器 padding 決定 section 位置，導致書頁無法真正貼右
- 沒有另外包內容 inner wrapper，結果內容視覺上被做成置中

### 正確方向

- 小日獸圖鑑仍屬於既有 overlay 內的功能，不是獨立 route
- 書頁圖只是一層背景，要直接露出上邊與左邊
- 書頁 section 先貼齊應該對齊的外邊
- 內容再放進 inner wrapper，刻意讓出左側空間給裝訂線與背景質感
