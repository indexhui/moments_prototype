# 青蛙日記揭露拼圖流程規格

整理日期：2026-07-03
補充日期：2026-07-08
地點紙膠帶流程修訂：2026-07-11

本文件是 `bai-entry-2`「搬家」不完整日記的 canonical 規格。此流程要逐步接近直太郎 / 黃金獵犬日記揭露拼圖的體驗：玩家先看到不完整日記，完成拼圖或照片回收後，日記頁再依序補完整。

本文件刻意拆開兩個很容易混淆的節點：

- `initial`：黃金獵犬後、尚未拍到青蛙前。第一頁拼圖完成後只拿到 `便利商店` 線索。
- `first-photo`：第一次在便利商店拍到青蛙後。先補完第一頁後半段，再按「繼續」進第二頁圖層拼圖。

## 需求背景

這次需求一開始是「目前青蛙長這樣，要逐漸將青蛙改成像黃金獵犬那樣，圖片先用 `diary_02_01`」。這句的重點不是只換圖片，也不是把青蛙日記直接變成一張完整插圖，而是要參考黃金獵犬的日記揭露拼圖流程。

過程中曾出現幾個錯誤理解：

- 只把空白框換成 `diary_02_01`，沒有建立拼圖流程。
- 做出拼圖外觀，但拼圖無法移動。
- 完成拼圖後按「繼續」，畫面又回到舊版殘缺日記頁。

最終確認後的規格是：流程順序必須像黃金獵犬日記揭露拼圖，不是先拍到青蛙照片，也不是先給完整圖片。

## 初始線索流程

這段是黃金獵犬後、尚未拍到青蛙前的 `initial` 流程。

1. 玩家從青蛙的未還原日記入口打開 `bai-entry-2`。
2. 不先顯示第一頁日記，先進入獨立的滿版「紙膠便簽_書籤」視覺解謎；這個畫面要照 Figma frame 思考，不再放進日記頁小框。
   - 不顯示棕色標題列。
   - 不顯示說明膠囊或教學橫條。
   - 紙膠解謎 frame 本身要吃滿可用版面。
3. 不使用文字線索、語意推理、選字拼字，也不顯示三個 `?` 收納槽與「還差幾組」。
4. 紙面必須被大量文字與花紋紙膠帶填滿，不留大片無內容區。包含完整地點紙膠帶 `便利商店`、`街道`、`甜點店`，也混入花紋紙膠與其他地點干擾項，例如 `天橋`。
5. 玩家拿到一張可自由平移的白底棕框長書籤；書籤上有三個固定膠痕輪廓，也同時是最後黏貼紙膠帶的三個空位。三個膠痕分別是長條、短方、缺角中長形，且膠痕的寬高必須與對應答案紙膠帶一致。
6. 玩家可以把任何一張紙膠帶撕起來拖動。正確的三張是「完整地點文字」且形狀剛好吻合書籤膠痕；其他紙膠帶不是地點，或雖然像地點但已殘破、破口無法覆蓋膠痕。
7. 錯誤紙膠帶也能被撕起，但拖到書籤空槽時不會貼牢，會短暫震動並滑回原位。正確紙膠帶必須親手拖到對應形狀的膠痕中黏好；放錯形狀也會回到原處。
8. 三張正確紙膠帶全部貼在書籤上後，才算一次取得 `便利商店`、`街道`、`甜點店`，並顯示「翻開日記」。
9. 玩家按「翻開日記」後，第一頁才出現：
   - 標題是 `???`。
   - 插圖區是可移動的四片直向拼圖。
   - 文字區是被拼圖影響的文字格，不是舊版灰色破損長條頁。
10. 第一頁剛打開時不得顯示地點書籤或地點紙膠帶；玩家只能先看到 `???`、插圖拼圖與被拼圖影響的文字格。
11. 玩家拖曳或點選交換拼圖片，把插圖還原。
12. 拼圖未完成時，地點書籤與地點紙膠帶都不可出現；也不得額外畫出句子空格紙片。
13. 拼圖完成後，前面取得的實體地點書籤才以絕對定位浮在書頁下方；書籤樣式、比例、三個膠痕與紙膠帶必須和取得階段一致：
   - `便利商店` 紙膠帶仍貼在書籤上。
   - `街道` 紙膠帶仍貼在書籤上。
   - `甜點店` 紙膠帶仍貼在書籤上。
   - 書籤預設停在正文下方偏右，不得壓住最後一行文字；玩家可拖曳書籤本體，在書頁可用範圍內自由移動。
14. 句子中的 `＿＿＿＿` 直接換成與 `便利商店` 紙膠帶完全同形、同尺寸的虛線輪廓；輪廓嵌在正文位置，不另畫獨立空格紙片。玩家必須從書籤撕起 `便利商店`，拖進正文輪廓後才補字。撕起時紙膠帶本體要離開書籤並跟手移動，書籤原位同步露出同形膠痕；放錯或放開在空白處才回到原位。
15. `便利商店` 填入後，日記文字才補成完整句子；書籤上的便利商店位置留下同形膠痕，跨頁視為已使用。
16. 完成紙膠帶填入後，頁面才出現「繼續」。
17. 玩家按「繼續」後，不切回舊版日記頁，也不揭露後半段文字，直接在同一個拼圖日記頁上顯示線索 overlay。
18. 線索 overlay 顯示：
   - 標題：`獲得線索`
   - 線索：`便利商店`
19. 玩家按 overlay 的「繼續」後，結束這次日記揭露流程並回到上一層。

這段流程的核心是：

```text
撕起任意紙膠帶 -> 觀察是否為完整地點與是否吻合書籤膠痕 -> 錯誤或殘破膠帶貼不上會滑回 -> 親手撕下三張正確膠帶 -> 分別貼進對應形狀的書籤三槽 -> 帶著完成書籤翻開日記 -> 完成拼圖 -> 親手填入「便利商店」 -> 獲得線索「便利商店」
```

這裡不得出現 `FROG_MOVING_DIARY_FRAGMENT.revealText`，也就是不得提前顯示：

```text
但小白的表情一臉問號的看著我。
外面有一陣騷動打亂我的不安....
```

## 第一次拍到青蛙後的頁序

這段是便利商店事件第一次拍到青蛙後的 `first-photo` 流程。此時照片只讓青蛙在日記裡留下剪影，玩家還沒有正式收集青蛙。

前置順序：

1. `FrogDiaryClueEventModal` 播旁白：`青蛙發現被拍下後直接跳走了。`
2. `ArrangeRouteView` 記錄第一次青蛙照片，`streetForgotLunchFrogPhotoAttemptCount` 變成 `1`。
3. `DiaryOverlay` 用 `mode="frog-fragmented-diary"` 打開。
4. 先跑 `PhotoDiarySlidePage` / `firstPhotoSlideAcross`，讓青蛙照片飛進日記。
5. 回到剪影頁，問號變成青蛙剪影。
6. 小麥與小貝狗說明青蛙還沒被收集，只是在日記留下剪影，接著引導玩家查看日記。

日記頁序：

1. 先顯示第一頁的黃金獵犬式補完頁，也就是 `BaiEntry2MovingDiaryRevealPage`。
2. 第一頁標題先是 `???`。
3. 第一頁圖片仍是 `diary_02_01.jpg`，保留問號拼圖修復的視覺節奏。
4. 第一頁前半段 `openingText` 要一開始保留：

```text
今天和小麥請搬家公司搬家。
整理到一半，客廳出現幾瓶便利商店飲料，
我以為是小麥買的，就很自然地全部喝掉了。
```

5. 第一頁後半段 `revealText` 在揭露時間點才浮出：

```text
但小白的表情一臉問號的看著我。
外面有一陣騷動打亂我的不安....
```

6. 後半段浮出時使用青綠色揭露調性，標題解析完成後沉回一般日記色。
7. 標題從 `???` 解析成 `搬家` 後，才顯示「繼續」。
8. 玩家按「繼續」後，才進第二頁。
9. 第二頁才顯示 `BaiEntry2StreetPuzzlePage`，開始圖層拼圖。
10. 第二頁圖層拼圖未完成前，文字使用 `secondPuzzlePromptText`，地點保留兩格空缺：

```text
跑出去看，才發現＿＿亂成一團。
原來有人在玩球，撞到了正在發傳單的人。
```

11. 第二頁一開始不得顯示地點書籤或地點紙膠帶；玩家必須先完成四層圖片拼圖。
12. 四層圖片拼圖完成後，同一張地點書籤才再次以絕對定位浮在書頁下方，不重新生成選項，也不重播取得演出：
    - `便利商店` 已使用。
    - `街道` 尚未使用。
    - `甜點店` 尚未使用。
    - 書籤仍可拖曳移動，初始位置須避開第二頁正文。
13. `＿＿` 只保留為正文缺字結果，不另畫可拖放的空格紙片。
14. 第二頁正文的 `＿＿` 直接換成與 `街道` 紙膠帶完全同形、同尺寸的虛線輪廓。玩家必須從書籤撕起 `街道` 並拖進正文輪廓；放入其他紙膠帶時只顯示不相符回饋，不扣除進度。
15. 放入 `街道` 後，文字才補成完整的 `secondPuzzleText`：

```text
跑出去看，才發現街道亂成一團。
原來有人在玩球，撞到了正在發傳單的人。
```

16. 書籤狀態更新為便利商店與街道已使用，甜點店仍貼在書籤上。
17. 完成紙膠帶填入後才顯示「繼續」。
18. 玩家按第二頁「繼續」後，才開 `FragmentedDiaryClueOverlay`。
19. overlay 顯示：
    - 標題：`獲得提示`
    - 提示：`街道`

這段流程的核心是：

```text
第一次青蛙照片 -> 第一頁後半段浮出 -> 繼續 -> 第二頁圖層拼圖 -> 使用保留在書籤上的「街道」紙膠帶 -> 獲得提示「街道」
```

## 第二次拍到青蛙後的頁序

這段是街道事件第二次拍到青蛙後的 `second-photo` 流程。

1. 仍先走照片飛入與剪影更新。
2. 不要回頭重播 `initial` 的便利商店拼圖，也不要重播第一次照片後的第二頁圖層拼圖。
3. 先顯示第二頁的黃金獵犬式補完頁，流程與第一次青蛙照片後的第一頁補完相對應：

   - 已恢復的前段 `跑出去看，才發現街道亂成一團。` 一開始就保留。
   - 後段 `原來有人在玩球，撞到了正在發傳單的人。` 到揭露時間點才以青綠色浮出。
   - 標題先顯示 `???`，最後解析成 `搬家`，文字同時沉回一般日記色。

4. 玩家按第二頁「繼續」後，才進第三頁圖片拼圖；第三頁暫用 `diary_02_03.png`。

```text
忙完了一天，終於能繼續搬家。
```

8. 玩家必須把書籤上的 `甜點店` 紙膠帶拖進正文同形輪廓，文字才補成完整內容，書籤進度也更新為三張皆使用。

這段流程的核心是：

```text
第二次青蛙照片 -> 第二頁前段保留、後段浮出 -> 標題恢復 -> 繼續 -> 第三頁圖片拼圖 -> 使用書籤上的「甜點店」紙膠帶 -> 獲得提示「甜點店」
```

## 絕對不可改錯的順序

- `initial` 不是先拍到青蛙照片。
- `initial` 不是先顯示完整 `diary_02_01` 插圖。
- `initial` 不是先閱讀舊版殘缺日記再拿線索。
- `initial` 完成拼圖後不得跳到舊版 `1/3` 缺字頁。
- `initial` 完成拼圖後不得顯示第一頁後半段文字，只能給 `便利商店` 線索。
- `initial` 必須先跑書籤膠痕比對、撕取、黏貼三張紙膠帶的完整解謎；第一頁圖片與正文不可提早出現。
- 地點紙膠帶必須使用取得階段同一張書籤與同一組膠帶，不可退回大型彩色卡片或整理好的選項面板。
- 不可顯示頂部三個 `?` 收納槽或「還差幾組」。
- 不可使用文字線索、語意推理、選字或依序拼字；正確膠帶由「完整地點文字」與「書籤固定膠痕形狀」共同辨識。
- 紙面必須由大量有文字的紙膠帶填滿，不可退回稀疏散字、規則字盤或只有三個答案選項。
- 書籤必須是獨立可移動物件，三個膠痕輪廓也必須是實際收納槽；不可點一下答案就自動收取。
- 書籤膠痕不可只是比例提示；每個膠痕的實際寬高必須等於對應答案紙膠帶，貼上後不能縮放跳動。
- 玩家必須完成「撕起膠帶、比對膠痕形狀、拖到書籤、放入對應空槽」四個操作。錯誤膠帶可以被撕起，但不能貼牢，錯誤落點不可消耗膠帶。
- 「翻開日記」只能在三張正確紙膠帶都貼到書籤後出現。
- 收下後，三組地點紙膠帶不可立刻顯示；必須等各頁拼圖完成後，才讓同一張地點書籤本體浮在書頁下方絕對位置。
- 日記頁中的書籤本體必須可自由拖曳，並限制在書頁範圍內；預設位置與正文要保留完整間距，不可蓋住文字格。
- 拼圖完成只負責讓書籤浮出並開放正文輪廓；玩家必須親手把正確紙膠帶從書籤拖進正文同形輪廓，才算完成該頁。
- 已使用紙膠帶必須跨頁保留成同形膠痕，不能再次選取。
- `first-photo` 不得直接跳第二頁圖層拼圖；必須先顯示第一頁後半段文字浮出。
- `first-photo` 的第二頁圖層拼圖必須等第一頁「繼續」後才出現。
- 不是只做靜態切圖；拼圖必須能拖曳，也必須能點選交換。

## 拼圖規格

- 拼圖區使用 `diary_02_01.jpg`。
- 圖片比例：`640 / 460`。
- 拼圖切成 4 片直向切片。
- 正確順序：`[0, 1, 2, 3]`。
- 初始打亂順序：`[2, 0, 3, 1]`。
- 第三片拼圖必須是問號片。
  - 目前對應 `pieceId = 2`。
  - 在正確順序中，它會位於第三格。
- 玩家可以：
  - 拖曳拼圖片換位。
  - 點選第一片、再點第二片來交換。
- 拼圖未完成時不顯示地點書籤，也不顯示底部「繼續」。
- 拼圖完成後只浮出書籤本體與正文內嵌的同形輪廓；正確紙膠帶拖入正文輪廓後才顯示底部「繼續」。

## 文字與地點紙膠帶規格

文字區要參考黃金獵犬揭露拼圖的文字格，而不是舊版破損長條文案。

- 文案以單字元文字格呈現。
- 地點關鍵字不再用工具列呈現；拼圖完成後只浮出同一張實體書籤，玩家直接撕取書籤上的紙膠帶。
- 地點書籤必須用絕對定位浮在書頁下方，不佔用圖片區上方排版空間。
- 文字格會依拼圖片位置散落或回到正確位置。
- 拼圖越接近正確順序，文字越接近可讀狀態。
- `便利商店` 在拼圖完成前以 `＿＿＿＿` 保留；拼圖完成後改顯示與該紙膠帶同形、同尺寸的正文輪廓，玩家拖入紙膠帶後才補回正文並對應到線索。
- 不使用舊版 `[[...]]` 破損遮罩作為此第一段揭露拼圖的主畫面。

## 線索揭露規格

`initial` 完成拼圖並填入 `便利商店` 紙膠帶後，按「繼續」時必須直接開啟線索 overlay。

線索 overlay：

- 使用既有 `FragmentedDiaryClueOverlay`。
- `headingText` 維持預設 `獲得線索`。
- `clueText` 是 `便利商店`。
- 不顯示舊版頁碼 `1/3`。
- 不重新 render 舊版 `buildBaiEntry2FragmentPages("initial")` 頁面。

這次修正的關鍵是：`continueAfterBaiEntry2Puzzle` 不能只把 `hasCompletedBaiEntry2Puzzle` 設為 `true`，因為那會讓 render 條件跳出拼圖頁並回到舊版 initial 殘缺日記頁。正確做法是，在引導流程中直接呼叫 `startFragmentedDiaryClueReward()`，讓線索 overlay 留在拼圖頁上揭露。

## 實作守則

- `continueAfterBaiEntry2Puzzle` 只服務 `initial` 第一頁拼圖完成後的便利商店線索，不得用它觸發第一次青蛙照片後的文字補完。
- `continueAfterBaiEntry2Puzzle` 必須同時檢查拼圖完成與 `便利商店` 紙膠帶已填入。
- `hasAcceptedBaiEntry2LocationTiles` 必須加入主 `content` 的 dependency list，避免按「收下並翻開日記」後畫面不更新。
- `baiEntry2InitialLocationId` 與第二頁的紙膠帶狀態要分開保存，但畫面上要呈現成同一張書籤的連續使用狀態。
- 第一次青蛙照片後的第一頁補完使用 `shouldPlayBaiEntry2FirstPhotoReveal`。
- 第一次青蛙照片後，玩家按第一頁「繼續」時只應設定本地狀態，例如 `hasAdvancedBaiEntry2FirstPhotoReveal`，再讓 render 進入 `BaiEntry2StreetPuzzlePage`。
- 第一頁補完與初始拼圖共用 `BaiEntry2MovingDiaryRevealPage` 的視覺語彙，但兩者的觸發時機與後續動作不同。
- 不要把 `openingText`、`revealText`、`firstText` 合併成單一字串來控制流程。前半段要保留，後半段要在揭露時間點才浮出，完整頁才使用 `firstText`。
- 新增日記狀態、continue handler、或 reveal flag 時，必須同步補進 `DiaryOverlay` 主 `content` `useMemo` 的 dependency list，避免點擊後畫面卡在舊 props。

## Hibimon 入口規格

在 `/hibimon` 頁面，青蛙未還原日記入口要依 `unrestoredFrogFragmentPhotoAttemptCount` 預覽對應階段。

條件：

- `variant === "unrestored"`
- `target.unrestoredView === "entry-bai-2-fragment"`

對應規則：

- `target.unrestoredFrogFragmentPhotoAttemptCount === 0`
  - 預覽 `initial`。
  - 第一頁拼圖完成後只顯示 `獲得線索 / 便利商店`。
  - `DiaryOverlay` 使用 `mode="frog-diary-catalog-guide"`。
- `target.unrestoredFrogFragmentPhotoAttemptCount === 1`
  - 預覽 `first-photo`。
  - 先顯示第一頁後半段文字浮出，按「繼續」後才顯示第二頁圖層拼圖。
- `target.unrestoredFrogFragmentPhotoAttemptCount === 2`
  - 預覽 `second-photo`。
  - 第一頁已完整，第二頁補完，並出現下一段殘缺線索。

所有入口都使用 `initialJournalView="entry-bai-2-fragment"`，並在流程結束後用 `onFragmentedDiaryComplete={onClose}` 回到 Hibimon 表格。

## QA 檢查表

每次改此流程後，至少要驗證：

`initial`：

1. `/hibimon` 打開青蛙第一個未還原日記。
2. 第一個畫面只能顯示「找出三張地點紙膠帶」解謎，不得出現第一頁圖片、正文或 `???`。
3. 畫面不得出現三個 `?` 收納槽、「點一下散落文字格」或「還差幾組」。
4. 紙面要被大量文字紙膠帶填滿，且混有非地點碎片與破損地點膠帶。
5. 書籤是獨立可移動物件，書籤上三個槽必須呈現不同膠痕輪廓，且膠痕大小與答案紙膠帶一致。
6. 任意紙膠帶都能被撕起拖動；錯誤或殘破紙膠帶拖到書籤後會顯示回饋並滑回原位。
7. `便利商店`、`街道`、`甜點店` 必須分別貼到對應形狀的膠痕槽中，貼錯形狀不可消耗。
8. 三張正確紙膠帶全部貼牢前不得出現「翻開日記」。三張完成後才出現。
9. 按「翻開日記」後，才出現 `???` 與四片直向拼圖。
10. 第三片拼圖是 `?`，拼圖可以拖曳，也可以點選交換。
11. 拼圖未完成時沒有地點書籤，也沒有底部「繼續」。
12. 拼圖完成後，同一張地點書籤才浮在書頁下方絕對位置；不額外顯示句子空格紙片，且尚未出現「繼續」。
13. 拖動書籤上的 `便利商店` 紙膠帶到正文內同形輪廓，才可以補成完整句子；單點只選取，不會直接作答。
14. 填入後便利商店位置留下同形膠痕且不可再次選取，並出現「繼續」。
15. 按「繼續」後直接顯示 `獲得線索 / 便利商店`。

`first-photo`：

2. 問號剪影頁出現後，再進搬家日記。

## 給後續實作者的提醒

當需求說「參考黃金獵犬」時，優先複製的是流程與互動節奏，其次才是視覺細節。這裡的玩家認知順序比單張圖片更重要：玩家必須先意識到日記不完整，透過拼圖修復它，最後才得到下一個地點關鍵字。
