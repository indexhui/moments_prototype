# 日記線索引導版本

本文件記錄 2026-06-04 版本的「日記線索引導」概念與流程。此版本把交換日記從單純的解鎖頁面，改成會承接小日獸捕捉、日記修復、下一篇線索、安排路線的連續體驗。

2026-07-03 補充：青蛙 `bai-entry-2` 第一段不完整日記已改為「先看不完整日記 -> 完成拼圖 -> 獲得線索」的黃金獵犬式揭露拼圖流程。完整規格以 `docs/FROG_DIARY_REVEAL_PUZZLE_FLOW.md` 為準。

2026-07-08 補充：青蛙日記要分清楚 `initial` 與 `first-photo`。黃金獵犬後的 `initial` 第一頁拼圖完成後只拿到 `便利商店` 線索；第一次拍到青蛙後的 `first-photo` 必須先顯示第一頁後半段文字浮出，再按「繼續」進第二頁圖層拼圖。

## 設計概念

- 日記不是一次直接完整打開，而是有「未知、殘缺、完整」三種狀態。
- 玩家拍到小日獸後，照片仍會進入對應小日獸內頁；原本的照片飛入流程要保留。
- 完整日記的揭露需要有儀式感：先看到第一格，再修復文字，玩家按下一頁後才進第二格，最後才進入角色對話。
- 下一篇日記會先以殘缺線索出現，提示玩家接下來要透過安排路線去找新的小日獸或事件。
- 玩家在引導流程中點日記時，不用依賴頁面中央按鈕；主要由下方對話 UI 推進。

## 容易失誤的實作守則

這段是給後續維護用的硬規則。若要改青蛙線或新增類似小日獸 reveal，先看這裡。

### 照片飛入儀式

- 小日獸照片「跑進日記本」不是普通貼上動畫。
- 必須復用直太郎已存在的照片飛入流程：
  - `PhotoDiarySlidePage`
  - `firstPhotoSlideAcross`
  - `firstPhotoDiaryStage === "photo-slide"`
- 不要改成局部卡片的 `revealStageIn`、`polaroidStickIn`、或自製簡化版貼上動畫。
- 青蛙第一次線索雖然不會直接收集成功，也仍然要先走完整照片滑入，再進剪影頁。

### 青蛙第一次線索的狀態順序

青蛙第一次拍照後的正確順序：

1. `FrogDiaryClueEventModal` 播旁白：`青蛙發現被拍下後直接跳走了。`
2. `ArrangeRouteView` 記錄第一次青蛙照片與 `streetForgotLunchFrogPhotoAttemptCount`。
3. 開 `DiaryOverlay`，mode 使用 `frog-fragmented-diary`。
4. `DiaryOverlay` 先進 `firstPhotoDiaryStage = "photo-slide"`。
5. 顯示 `PhotoDiarySlidePage`，跑 `firstPhotoSlideAcross`。
6. 回到 `FrogFragmentPhotoIntroPage`，問號改成青蛙剪影。
7. 小麥與小貝狗用三句對話說明：青蛙還沒被收集起來，只是在日記裡留下剪影，接著看看這篇日記。
8. 進入搬家殘缺日記第一頁補完頁。
9. 第一頁保留原本前半段，後半段「但小白的表情...外面有一陣騷動...」在揭露時間點才浮出。
10. 玩家按第一頁「繼續」後，才進第二頁圖層拼圖。
11. 第二頁圖層拼圖完成後，才顯示 `獲得提示 / 街道`。

### 殘缺日記閱讀方式

- 青蛙線的搬家殘缺日記要用一則一則的 slide。
- `VisualDiaryBookPage` 需使用：
  - `pageMode="slide"`
  - `slideTotalPages={3}`
- 不要讓玩家在同一頁裡縱向滑動看多則日記。
- 第一次拍到青蛙後不得直接跳第二頁；必須先讓第一頁後半段浮出。
- 其他完整日記或舊流程若本來就是 scroll/restoration，可以維持原本形式。

### 不要直接揭露系統機制

- 不顯示 `1/3`、`2/3` 這類收集進度教學。
- 不對玩家說「還不會立刻收集成功」。
- 線索要透過照片、剪影、殘缺日記和角色對話傳達，不用系統說明把規則攤開。

## 第一篇殘缺日記讀後線索

玩家第一次讀到「滑雪板搭捷運」的殘缺日記時，不會只靠小麥對話收尾，而是要有一次遊戲式線索獲得演出。

流程：

1. 玩家讀完第一篇殘缺日記。
2. 小麥說「僅存的日記也只有一點點就斷掉了」。
3. 玩家點下方對話 UI 的「點擊繼續」。
4. 日記畫面蓋上暗幕，文字提示「小日獸會出現在日記提到的人、事、物」。
5. 提示文字約 2.5 秒淡入淡出。
6. 顯示「獲得線索」回饋：`安排行程時經過捷運`。
7. 回到原本流程，帶玩家進入下一段安排路線。

## 直太郎與第一篇完整日記

直太郎是拍到一次就會完成的小日獸，因此牠會直接還原第一篇日記。

流程：

1. 玩家拍到直太郎。
2. 照片進入直太郎內頁，保留原本「照片飛入」與收集到小日獸的流程。
3. 玩家在直太郎內頁按下「還原完整的日記」。
4. 進入對應日記頁，標題先顯示 `???`。
5. 第一格日記淡入，文字先呈現和先前殘缺日記一致的遮蔽內容。
6. 第一格文字用補字效果在 `1/2` 內從遮蔽狀態還原成完整句。
7. 第一格完整後，標題從 `???` 變成「雪板卡住」，並停在 `1/2`。
8. 玩家按「下一頁」後才切到 `2/2` 第二格。
9. 玩家在第二格按「繼續」後，小麥開始說話。
10. 玩家讀完小麥對話後，回到交換日記目錄。

第一篇目前內容：

- 標題：雪板卡住
- 第一格：帶著滑雪板要去滑雪，結果上捷運時，發現大家都在看我，才發現車門一直關不起來。
- 第二格：轉頭一看才發現，自己的板頭有一半露在車廂外，車門才會一直夾到關不起來。

## 第一篇還原後的第二篇顯現

玩家讀完第一篇完整日記後，會先回到交換日記目錄頁，直接進行下一篇日記透露。
第二篇透露完成後，才回到白天通勤收尾並接續離開捷運。

目前演出：

1. 玩家讀完第一篇完整日記的讀後對話。
2. 回到交換日記目錄頁。
3. 目錄中第二篇「搬家」以浮現動態出現，小麥說「第二篇日記出現了。」
4. 手指引導玩家點擊第二篇日記卡片。
5. 打開第二篇後，小麥說「但是一樣是不完整的。」
6. 玩家點下方對話 UI 的「點擊繼續」後，對話消失，完整露出第二篇第一格不完整日記。
7. 第二篇第一格不完整日記先顯示可操作的四片直向拼圖與文字格；玩家必須完成拼圖，頁面才顯示「繼續」。
8. 玩家完成拼圖並點擊「繼續」後，直接在同一頁顯示「獲得線索」：`便利商店`。
9. 線索 overlay 也使用明確的「繼續」按鈕關閉，不再自動結束或整片點擊結束。
10. 不再重複第一篇的「小日獸會出現在日記提到的人、事、物」規則教學。
11. 關閉日記流程，接回捷運事件收尾，繼續離開捷運。

## 搬家殘缺日記

搬家篇目前是下一段探索線索，不是完整篇章。

呈現規則：

- 標題顯示 `???`。
- 只顯示第一格。
- 圖片使用 `public/images/diary/diary_02_01.jpg`。
- 圖片區是四片直向可操作拼圖，第三片是問號片。
- 文字區使用黃金獵犬日記揭露拼圖式的文字格；拼圖修復時文字格也回到可讀位置。
- 玩家必須先完成拼圖，才會看到頁面底部「繼續」。
- 完成拼圖後按「繼續」直接揭露線索 `便利商店`，不得回到舊版 `1/3` 破損長條頁。

目前文字：

```text
今天和小麥請搬家公司搬家。整理到一半，客廳出現幾瓶便利商店飲料，我以為是小麥買的，就很自然地全部喝掉了。
```

關鍵字是 `便利商店`。詳細互動與 QA 規格見 `docs/FROG_DIARY_REVEAL_PUZZLE_FLOW.md`。

## 安排路線銜接

第一次出門流程已改成「安排路線」方式，而不是直接選交通方式。

目前方向：

- 畫面引導玩家把下方拼圖拉到路線空格，做出今天的出行決定。
- 有 modal 教學演出，示範拼圖從下方移到空格。
- 日記按鈕會出現在安排路線畫面上，暫時使用「地上的筆記本」作為按鈕圖。
- 在安排路線中點開日記時，要有返回按鈕，位置在左下角。
- 在安排路線流程中點日記，返回應直接回到安排路線。

搬家殘缺日記後的青蛙線目前主線：

- 第一次進入青蛙便利商店線索前，不再由一般安排路線直接安排便利商店。
- 玩家會先在上班中午觸發「忘記帶便當」對話，接著進入特殊 `公司 -> 便利商店` 路線關卡。
- 特殊關卡規格記錄在 `docs/WORK_LUNCH_CONVENIENCE_ROUTE_LEVEL.md`。
- 特殊關卡完成後，進入青蛙線索事件。
- 玩家安排其他地點時，維持該地點日常事件。
- 便利商店事件本身直接從店內「微波涼麵」青蛙線索與拍照流程開始。
- 舊版街道前置「在前往便利商店的路上 / 忘記帶便當」已移出青蛙事件資料，改由上班中午對話與特殊路線關卡承接。
- 青蛙線需要拍照三次才會還原完整日記：
  - 第一次拍照後，事件先播旁白 `青蛙發現被拍下後直接跳走了。`，再顯示青蛙照片飛入日記本、問號轉成剪影與殘缺日記 reveal；照片飛入必須復用直太郎照片的 `PhotoDiarySlidePage` / `firstPhotoSlideAcross` 儀式感動態，不要改成簡化版貼上動畫。剪影出現後，小麥與小貝狗先用三句對話說明青蛙還沒被收集起來、只是留下剪影，並引導玩家查看這篇日記。殘缺日記改為一則一則 slide，不使用縱向滑動。完成後回到事件，小麥說 `忘記帶便當還能遇到小日獸，真是小確幸。`，再說 `趕緊回到公司享用涼麵吧。`，最後接回公司上班過場並直接下班。
  - 第二次拍照後，仍用同一套 reveal UI，顯示第一格恢復、第二格補完與第三格殘缺線索。
  - 第三次拍照後，才進入完整日記還原與讀後對話。
- 搬家殘缺日記不要另寫一套 UI；需透過共用的書頁 reveal 模組組裝頁面資料。
