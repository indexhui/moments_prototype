# 日記線索引導版本

本文件記錄 2026-06-04 版本的「日記線索引導」概念與流程。此版本把交換日記從單純的解鎖頁面，改成會承接小日獸捕捉、日記修復、下一篇線索、安排路線的連續體驗。

## 設計概念

- 日記不是一次直接完整打開，而是有「未知、殘缺、完整」三種狀態。
- 玩家拍到小日獸後，照片仍會進入對應小日獸內頁；原本的照片飛入流程要保留。
- 完整日記的揭露需要有儀式感：先看到第一格，再修復文字，再滑出第二格，最後才進入角色對話。
- 下一篇日記會先以殘缺線索出現，提示玩家接下來要透過安排路線去找新的小日獸或事件。
- 玩家在引導流程中點日記時，不用依賴頁面中央按鈕；主要由下方對話 UI 推進。

## 直太郎與第一篇完整日記

直太郎是拍到一次就會完成的小日獸，因此牠會直接還原第一篇日記。

流程：

1. 玩家拍到直太郎。
2. 照片進入直太郎內頁，保留原本「照片飛入」與收集到小日獸的流程。
3. 玩家在直太郎內頁按下「還原完整的日記」。
4. 進入對應日記頁，標題先顯示 `???`。
5. 第一格日記淡入，文字先呈現殘缺句。
6. 第一格文字用補字效果從殘缺變完整。
7. 第一格完整後，停一拍，再自然往上滑出第二格。
8. 第二格出現後，標題才從 `???` 變成「雪板卡住」。
9. 約一秒後，小麥開始說話。
10. 玩家讀完小麥對話後，回到交換日記目錄。

第一篇目前內容：

- 標題：雪板卡住
- 第一格：帶著滑雪板要去滑雪，結果上捷運時，車門一直關不起來。
- 第二格：轉頭一看才發現，自己的板頭有一半露在車廂外，車門才會一直夾到關不起來。

## 回到目錄後的第二篇顯現

玩家讀完第一篇完整日記後，會回到交換日記目錄頁。

目前目錄演出：

1. 第二則日記「搬家」顯現。
2. 小麥說：「第二則日記顯現出來了。」
3. 手指引導指向「搬家」這一篇。
4. 玩家點開後看到的是殘缺日記，而不是完整日記。

## 搬家殘缺日記

搬家篇目前是下一段探索線索，不是完整篇章。

呈現規則：

- 標題顯示 `???`。
- 只顯示第一格。
- 圖片先以空白框代替，等待美術補圖。
- 文字是「有內容但被破壞」的狀態，不直接用 `OO` 顯示，而是用破損文字塊呈現。
- 頁面中央不顯示「繼續」按鈕。
- 玩家點下方對話 UI 的「點擊繼續」來推進。

殘缺文字方向：

```text
OO那天，OO亂成一團。
紙箱堆得OO。看到客廳桌上有OO便利商店買回來的OO。
```

目前實作會把 `OO` 視覺化為破損文字塊，保留「原本有字但不完整」的感覺。

## 安排路線銜接

第一次出門流程已改成「安排路線」方式，而不是直接選交通方式。

目前方向：

- 畫面引導玩家把下方拼圖拉到路線空格，做出今天的出行決定。
- 有 modal 教學演出，示範拼圖從下方移到空格。
- 日記按鈕會出現在安排路線畫面上，暫時使用「地上的筆記本」作為按鈕圖。
- 在安排路線中點開日記時，要有返回按鈕，位置在左下角。
- 在安排路線流程中點日記，返回應直接回到安排路線。

## 後續注意

- 替換美術後，日記圖框比例先維持目前版型，不再回到舊的一張圖一頁。
- 殘缺日記的 `???` 標題要等完整還原後才改成正式標題。
- 完整日記的對話要在日記演出結束後才出現，不要提早打斷玩家閱讀。
- 若新增第三篇以後的線索，建議沿用「目錄顯現 -> 手指引導 -> 殘缺日記 -> 安排路線」的節奏。
