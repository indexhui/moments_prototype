# 上班中午便利商店路線關卡

本文件記錄「忘記帶便當後，從公司前往便利商店」這段特殊安排行程關卡的規格。若文件與程式不同，請以程式為準。

## 使用時機

- 入口在 `src/components/game/GameSceneView.tsx` 的上班場景 `scene-98-work`。
- 觸發條件：
  - 已解鎖第一篇完整日記 `bai-entry-1`。
  - 尚未解鎖第二篇完整日記 `bai-entry-2`。
  - 青蛙便利商店線索尚未拍照，`streetForgotLunchFrogPhotoAttemptCount === 0`。
  - 青蛙便利商店事件尚未完成。
  - 尚未觸發過 `hasTriggeredWorkLunchForgotBentoEvent`。
- 觸發後先播放上班中午對話：
  - 旁白：`中午時間。`
  - 小麥：`糟糕，今天忘記帶便當了。`
  - 小麥：`去便利商店買午餐好了。`
- 對話結束後進入 `/game/arrange-route?storyRoute=work-lunch-convenience`。

## 教學 Modal

教學 modal 是這一關開始前的「寬度銜接」示範，不代表實際地點。

規則：

- 只使用無地點路徑拼圖，不使用便利商店、街道等地點拼圖。
- modal 會在成功與失敗示範間輪播。
- 成功示範文字：`寬度一致的話可以連在一起`。
- 失敗示範文字：`寬度不一致，無法連接再一起`。
- 拖曳動畫需是一段順順的平移，不要分段折線移動。
- 拖曳拼圖到達空格後，拖曳中的那層拼圖要先消失，再顯示放入格子內的拼圖。
- 成功示範在放入後顯示綠色 `O`。
- 失敗示範在放入後顯示紅色 `X`，接縫紅線閃兩下後停留。
- 成功/失敗示範切換時，不要讓整個示範區閃一下；只重跑拖曳物件與結果狀態。

目前教學用拼圖：

- 正確上方拼圖：`/images/route/rt_010_010_111.jpg`
- 錯誤上方拼圖：`/images/route/rt_010_010_010.png`
- 固定下方拼圖：`/images/route/rt_1111_010_010.jpg`

## 關卡本體

實際關卡代表「公司 -> 便利商店」。

棋盤：

- 直向三格。
- 上方固定：便利商店地點拼圖，使用 `/images/route/route_new/wide_to_narrow_超商.png`。
- 中間空格：玩家放入路徑拼圖。
- 下方固定：公司起點拼圖，使用 `/images/route/route_new/wide_narrow_compnay.png`。

下方拼圖區：

- 玩家可拖拉的拼圖全部使用無地點路徑拼圖。
- 不顯示中文名稱。
- 四個拼圖要剛好排成一列，不需要橫向捲動。
- 目前順序：
  - `/images/route/rt_1111_010_010.jpg`：寬接窄
  - `/images/route/rt_010_010_010.png`：直路
  - `/images/route/rt_111_010_111.jpg`：寬接寬
  - `/images/route/rt_010_010_111.jpg`：窄接寬

## 寬度判斷

這一關只需要玩家銜接中間一格，但要同時檢查上下兩側接縫。

- 上方便利商店底邊需要接 `narrow`。
- 下方公司頂邊需要接 `wide`。
- 正確答案是 `narrow -> wide`，也就是 `/images/route/rt_010_010_111.jpg`。

目前四個候選拼圖的寬度資料：

- `wide-to-narrow-route`：top `wide`，bottom `narrow`
- `straight-route`：top `narrow`，bottom `narrow`
- `wide-to-wide-route`：top `wide`，bottom `wide`
- `narrow-to-wide-route`：top `narrow`，bottom `wide`

錯誤提示：

- 紅色提示要畫在兩塊拼圖之間的縫隙，不要畫在中間拼圖內。
- 上方接縫不對，只亮上方縫隙。
- 下方接縫不對，只亮下方縫隙。
- 兩邊都不對，上下縫隙都亮。
- 正確拼圖不顯示紅色接縫。

## 拖放互動

- 玩家可以從下方拼圖區點選或拖曳拼圖放到中間空格。
- 放錯時仍允許放入，但顯示錯誤接縫與提示文字。
- 按出發時，如果拼圖不正確，不出發並保留錯誤提示。
- 已放入中間格的拼圖可以再拖起來。
- 把已放入的拼圖丟到棋盤兩側空白處，會移除中間格拼圖。
- 把已放入的拼圖丟回下方拼圖區，也會移除中間格拼圖。
- 把已放入的拼圖拖回中間格或棋盤本體，不應誤觸移除。

## 完成後銜接

玩家放入正確拼圖並按出發後：

- 寫入 `hasTriggeredWorkLunchForgotBentoEvent = true`。
- 記錄一次 arrange route departure。
- 播放兩點式出發過場：公司 -> 便利商店。
- 過場後回到 `/game/arrange-route?eventId=<目前青蛙線索 stage>`。
- 青蛙便利商店事件直接從便利商店內開始，不再重播舊版「在街道上忘記便當」前置。

## 維護注意

- 教學 modal 和關卡本體要分開看：
  - 教學 modal：純路徑、無地點。
  - 關卡本體：固定起終點有地點，上方便利商店、下方公司；玩家拖拉的四塊仍是無地點路徑。
- 若替換拼圖素材，需同步更新寬度資料表。
- 若調整棋盤格距或尺寸，紅色接縫位置要跟著更新，確保紅線仍在兩格之間。
- 不要把下方四個可拖拉拼圖的中文 label 顯示出來；label 只作為程式內部判斷與輔助文字。
