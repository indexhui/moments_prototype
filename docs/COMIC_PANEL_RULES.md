# 漫畫格開發規範

這份文件是之後依照劇本開發時的前置規格。

目的只有一個：
先判斷「這一段該用哪一種漫畫格方法」，再開始寫 code。

## 目前已定調的做法

目前只有兩種正式定調的方法。

### 1. 單張漫畫格

用途：
- 劇本明確是「單張 comic 圖出現」
- 畫面重點是一張完整漫畫圖
- 不需要同時排兩張 panel

參考案例：
- 洗漱 / 起床上班那段 `freshen`

程式做法：
- 在 `GameSceneView.tsx` 走 `playStoryComic(...)`
- 對應圖資由 `COMIC_IMAGE_BY_ID` 管理
- 顯示方式是單張圖直接出現，不用 `storyComicOverlays`

關鍵位置：
- `src/components/game/GameSceneView.tsx`
- `playStoryComic(...)`

規則：
- 如果劇本寫的是單張 comic 圖，就只能先考慮這種
- 不要用 `storyComicOverlays` 去假裝做單張漫畫格

### 2. 雙張漫畫格

用途：
- 劇本明確是兩張 panel
- 需要右邊先出、左邊再出
- 需要兩張在同一段連續保留

參考案例：
- 袋子蠕動 / 小貝狗探頭那段

程式做法：
- 在 `scenes.ts` 使用 `storyComicOverlays`
- 每張 panel 分別定義位置、尺寸、進場方向、延遲
- 由 `GameSceneView.tsx` 的 overlay render 邏輯負責顯示

關鍵位置：
- `src/lib/game/scenes.ts`
- `src/components/game/GameSceneView.tsx`
- `storyComicOverlays`

規則：
- 只有劇本明確是雙 panel 時才用這種
- 不要把單張需求塞進這套

## 尚未定調的方法

以下目前都還沒有正式規範，不能自行發明做法：

- 一般場景物件
- 一般道具圖
- 「吐出一個道具」這種介於物件與漫畫格之間的演出
- 純 prop 圖但又想做成像漫畫格的畫面

現在的原則是：

- 如果劇本需求明確屬於單張漫畫格，就用單張漫畫格
- 如果劇本需求明確屬於雙張漫畫格，就用雙張漫畫格
- 如果不屬於這兩種，就先停下來確認，不要自己擴充第三種做法

## 開發前檢查清單

之後每次要照劇本往下做前，先依序檢查：

1. 劇本這一格是單張還是雙張？
2. 如果是單張：
   使用 `playStoryComic(...)`
3. 如果是雙張：
   使用 `storyComicOverlays`
4. 如果看起來像道具圖、場景物件、吐出物件：
   先視為「尚未定調」，不要直接實作
5. 只有在方法確定後，才開始補 scene、動畫、圖資

## 當前實作上的判斷基準

以下是目前已經確認過的分類：

- 袋子蠕動 / 小貝狗探頭：雙張漫畫格
- 洗漱 / 起床上班：單張漫畫格
- 相機：如果劇本定義它是漫畫格，優先視為單張漫畫格，不要先用雙張漫畫格實作

## 之後的工作方式

之後我在依照劇本開發前，應該先做這三件事：

1. 先讀這份文件
2. 先把該段需求歸類到「單張」或「雙張」
3. 若無法歸類，就先停下來確認，而不是直接實作
